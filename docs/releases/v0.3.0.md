## Highlights

- Translation behavior is now controlled from Settings.
- Supported languages now include auto, Chinese, English, Japanese, Korean, French, German, and Spanish.
- Target language no longer supports auto-detection, so output intent is explicit.
- Target-language options update based on the selected source language.
- Translation mode supports normal, natural, formal, and casual.
- Multi-language output can return English, Chinese, Japanese, and Korean sections.
- The floating panel result title, placeholder, hints, status copy, and primary labels follow the selected language.
- Non-English target prompts are stricter and explicitly tell the model not to fall back to English.
- The Settings floating-panel shortcut toggle section was removed to reduce clutter.
- Local Ollama and custom OpenAI-compatible API configuration remain supported.
- The GitHub project page now includes the app icon and a short product demo video.

## User Impact

Floatype can now be used for more than Chinese-to-English writing. For example:

- Chinese to Korean
- Chinese to Japanese
- English to Chinese
- Japanese to English
- French to English
- single-language output
- four-language output for comparison

The floating panel remains intentionally small: type, preview, edit, choose, insert.

Recommended pairing: use Floatype with 豆包输入法. 豆包输入法 handles everyday Chinese input and macOS IME behavior, while Floatype handles AI output language control and insertion.

## Notes

- Local Ollama users still need a running Ollama service and an installed model.
- Custom API users need an OpenAI-compatible chat completions endpoint.
- MLX support is reserved but not implemented.
- If Accessibility insertion is unavailable, Floatype copies the selected result to the clipboard for manual paste.
